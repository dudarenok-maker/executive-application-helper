# Length Check — word and character compliance skill

> **Setup note:** This is a template skeleton. The default bands below match the framework's `Project_Instructions.md` STEP 1 Length section — keep the two in sync if you tune them.

## When to invoke

- **Default trigger:** before saving any `.docx` to `[Workspace Folder Path]/Submissions/`, any screening-question response, or any short recruiter-response draft.
- **Explicit triggers:** "check the length", "count words", "verify the character count", "how long is this".

## Bands (defaults — matrix-row or brief-stated overrides win)

| Document type | Band |
|---|---|
| Cover Letter | 900–1,250 words |
| Statement of Claims | 1,000–1,250 words |
| Targeted Pitch | 600–1,000 words |
| Short recruiter response | strictly per the stated character / word limit |
| Platform screening question | **hard per-question character limit** (Seek: 1,000 chars; plain text only) |

## Method

1. Identify the document type and the operative band (brief-stated limit > matrix-row band > default band).
2. Count words per section and characters per screening answer. Count smart quotes, en/em-dashes as one character each.
3. Verdict per section: **pass / over / under**, with the delta and a recommended action (what to trim or where to add substance).
4. For an over-limit screening answer: recommend leading with the strongest one or two anchor proof points and dropping the rest — never deliver text the platform will truncate mid-sentence.

## Output and gate behaviour

Per-section table (section / count / band / verdict / action), then overall verdict. **Gate: band breaches block delivery until amended or the candidate explicitly accepts; hard platform character limits always block — no acceptance path** (silent truncation is worse than any trim).
