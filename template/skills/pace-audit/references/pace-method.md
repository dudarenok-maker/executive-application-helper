# Pace audit method — canonical reference (rev 2026-06-12, codifies the framework Conventions 9 and 11)

## The pace baseline

- **Do not assume 165 wpm** (typical executive baseline). Measure the candidate's actual pace at setup — many senior speakers run **~140–150 wpm including built-in pauses**, effectively **~130 wpm** where `[pause]` markers are expanded.
- Ask once at the start of any panel-prep session what the candidate's current pace is; default to the setup-measured value if unconfirmed.
- Worked example of why it matters: 525 words = ~3:11 at 165 wpm but ~3:37 at an actual 145 wpm.

## Counting rules

- Count spoken words only. Exclude: `[pause]` / `[short pause]` / `[STOP — pass back to panel]` markers, drop-list tables, headings, "Notes" placeholders.
- Add back pause time: ~2–3 sec per `[pause]`, ~1–2 sec per `[short pause]`.
- Runtime (sec) = words ÷ wpm × 60 + pause allowance.

## Budgets by format (defaults when intel is silent)

| Format | Budget per response |
|---|---|
| Recruiter screen | ~90 sec |
| Behavioural panel | 2–3 min |
| Executive panel | 2–3 min (with strategic framing) |

Recruiter/panel intel overrides defaults (e.g. "max 3 min per question, 4 questions in 30 minutes").

## Verdicts

| Verdict | Threshold | Action |
|---|---|---|
| Under | >10% under budget | Note headroom; no action |
| On | within ±10% | Pass |
| Over (advisory) | 10–25% over | Attach two-pass tightening plan |
| Over (gate) | >25% over | Block delivery until a tightening pass runs |

## The two-round tightening expectation (Convention 9)

When responses run over after a content reshape, plan for **two** passes:

1. **First pass — the drop list.** Cut the named drop-list items (parentheticals, repetition, supporting examples that don't carry weight). Recovers ~15–20 sec.
2. **Second pass — structural cuts the first pass didn't reach.** Compress result-block listings, remove one of two parallel-structure sentences, drop `[pause]` markers between sub-sections. Two passes together recover ~25–35 sec.

Single-pass tightening usually recovers only ~15–20 sec; budget the second pass from the start. **Never cut the closing principle lines** — they are the structural lands the panel remembers.

## Re-audit discipline (Convention 11)

Re-run the audit **after every reshape** — timing drifts with every edit, and the final-day pace recalibration (multi-stage iteration model, stage 4) depends on current numbers, not kickoff numbers.
