# Matrix-row traceability method — canonical reference (rev 2026-06-12, codifies the the framework matched-row mechanism)

## What a matrix row prescribes

The matched row (from `Examples_Section_5_1` … `5_8`) delivers all drafting attributes:

- **Positioning** — A-IDs with the dominant one flagged
- **Body stacks** — B-IDs grouped by paragraph
- **Template + length band** — C1–C6 and the word band (overrides the document-type defaults when present)
- **Fit-acknowledgment style** — how honest gaps are framed
- **AI register** — how AI capability is positioned
- **Companion files** — what loads

## Check mechanics

1. **Body-stack reflection.** For each B-ID in the stack, find the draft passage that carries it. "Reflected" means the entry's core response/proof points shape a paragraph — a one-clause allusion does not count. Record: B-ID → draft paragraph → reflected / thin / absent.
2. **A-ID dominance.** The dominant A-ID's framing should be recognisable in the opening third of the document. If a secondary A-ID's frame leads instead, that is drift even if all A-IDs appear.
3. **Template + length.** Take the band from the row's `Template/Length` column when present, else the document-type default. Use the `length-check` skill's count — do not recount.
4. **Fit-acknowledgment style.** Compare the draft's gap-handling passages to the row's prescribed style. Absence of any gap acknowledgment where the row prescribes one is drift.
5. **AI register.** Compare AI-related claims to the row's register; cross-check against Section 7.2 framing discipline (the `watchouts-sweep` skill covers breaches; this check covers register mismatch).
6. **Deviation accounting.** A deviation documented in the pre-drafting assessment (e.g. "dropping B2-8, brief gives it no hook") is legitimate. The finding category is *undocumented* drift.

## Verdicts

| Verdict | Meaning |
|---|---|
| **On-prescription** | All six checks pass |
| **Documented deviations only** | Deviations exist; all were named in the pre-drafting assessment |
| **Undocumented drift — review** | At least one undocumented deviation; [Candidate First Name] decides: regression (fix the draft) or improvement (keep, and update the matrix row at STEP 4 in the same session) |

## The row-calibration loop

Undocumented drift that survives review as an improvement is a matrix-row finding, not just a draft finding: amend the row (or its Notes column) at STEP 4 with a changelog entry, so the next draft against this archetype inherits the improvement. Drift that is never folded back is calibration leaking out of the system.
